## üìã Executive Summary

**MaCare** is a comprehensive digital healthcare platform designed specifically for maternal and child health management in Bangladesh. The system bridges the gap between healthcare providers (doctors and midwives) and expectant mothers, providing real-time health monitoring, expert medical advice, vaccination tracking, and emergency support throughout pregnancy and early childhood.

### üéØ Mission
To reduce maternal and infant mortality rates in Bangladesh by providing accessible, technology-driven healthcare solutions that connect mothers with healthcare professionals and empower them with knowledge and tools for healthy pregnancies.

### üë• Target Users
- **Expectant & New Mothers**: Primary users tracking pregnancy and child health
- **Doctors**: Medical professionals providing consultations and medical advice
- **Midwives**: Community health workers managing weekly maternal checkups

## üîß Technical Architecture

### Technology Stack

#### Frontend
- **Framework**: React 18+ with functional components
- **State Management**: React Hooks (useState, useEffect, useContext)
- **Routing**: React Router v6
- **Styling**: Tailwind CSS v3
  - Custom gradient utilities
  - Responsive breakpoints (sm, md, lg, xl)
  - Custom component classes
- **Icons**: Lucide React
- **Build Tool**: Vite
- **HTTP Client**: Axios
- **Authentication**: JWT with context API

#### Backend
- **Runtime**: Node.js v18+
- **Framework**: Express.js
- **Database**: MongoDB with Mongoose ODM
- **Authentication**: JWT (JSON Web Tokens)
- **File Storage**: Cloudinary (images, reports)
- **Email Service**: Nodemailer
- **Security**: 
  - bcrypt for password hashing
  - CORS configuration
  - Input validation and sanitization
  - Rate limiting

### API Architecture

#### RESTful Endpoints

**Authentication** (`/api/v1/users`)
- POST `/register` - User registration with role selection
- POST `/login` - JWT-based authentication
- POST `/logout` - Token invalidation
- POST `/renewaccesstoken` - Token refresh
- POST `/changepassword` - Password update

**Mother Features** (`/api/v1/mother`)
- GET `/dashboard` - Dashboard data
- POST `/maternal-record` - Create pregnancy record
- GET `/maternal-record` - Get pregnancy details
- PUT `/maternal-record/:id` - Update pregnancy info
- DELETE `/maternal-record` - End pregnancy (post-delivery)
- POST `/register-child` - Register newborn
- GET `/children` - List all children
- GET `/children/:id` - Child details
- POST `/kick-counter` - Log fetal movement
- GET `/kick-history` - Movement history
- GET `/doctor-advice` - All advice received
- PUT `/doctor-advice/:id/read` - Mark advice as read
- POST `/bp-tracking` - Log blood pressure
- POST `/weight-tracking` - Log weight

**Doctor Features** (`/api/v1/doctor`)
- GET `/dashboard` - Statistics and today's appointments
- POST `/search-mother` - Find patient by email
- GET `/mother/:email` - Complete patient profile
- POST `/send-advice` - Send medical advice
- GET `/my-advice` - All advice given
- GET `/advice-history/:motherId` - Patient-specific advice history
- GET `/patients` - All patients list
- PUT `/appointments/:id` - Update appointment status

**Midwife Features** (`/api/v1/midwife`)
- GET `/dashboard` - Assigned mothers and today's checkups
- GET `/assigned-mothers` - List all assigned mothers
- POST `/assign-mother/:motherId` - Assign new mother
- DELETE `/unassign/:motherId` - Remove assignment
- POST `/weekly-checkup` - Record checkup
- GET `/checkup-history/:motherId` - Mother's checkup history
- POST `/schedule-notification` - Set reminder
- GET `/today-checkups` - Today's scheduled checkups

**Common Features**
- GET `/profile` - User profile
- PUT `/profile` - Update profile
- POST `/upload-profile-picture` - Avatar upload
- POST `/upload-report` - Medical report upload
